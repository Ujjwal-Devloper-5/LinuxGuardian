# 🛡️ LinuxGuardian: Disaster Recovery Manual

This document is your **Ultimate Insurance Policy**. It contains the exact, step-by-step procedures to recover your data in any scenario—from accidentally deleting a single file to a total hardware failure.

## 🏗️ Scenario 1: The "Oops" Recovery
**Case:** You accidentally deleted or corrupted a specific file or folder, but your system is otherwise healthy.

### Using the One-Click Restore Utility
1. Open your terminal.
2. Run the interactive restore script:
   ```bash
   sudo linuxguardian-restore.sh
   ```
3. **Select Repository:** Choose **1** for Home (personal files) or **2** for System (OS configs).
4. **Browse Snapshots:** The tool will list all available versions. Copy the ID of the snapshot you want (e.g., `564cb824`).
5. **Set Target:** Choose a safe place to put the restored files (e.g., `/tmp/recovery`).
6. **Finalize:** Once the files appear in `/tmp/recovery`, manually copy the specific file you need back to its original location.

## 🌪️ Scenario 2: Total System Wipeout (Fresh Install)
**Case:** Your hard drive died, or your Arch Linux install is unbootable. You are starting with a blank machine.

### Step 1: Base Environment
Install a fresh version of Arch Linux. Once you have a terminal, install the core engines:
```bash
sudo pacman -S restic rclone
```

### Step 2: Re-link the Cloud "Ferrari"
You must re-connect to your 5TB Google Drive using your high-speed credentials.
1. Run `rclone config`.
2. Create a new remote named **`fortress-cloudbackup`**.
3. **CRITICAL:** Use your custom **Client ID** and **Client Secret** (saved in your private notes) to ensure MB/s speeds.
4. Complete the browser authentication.

### Step 3: The "Life" Restore (Home Directory)
This brings back your documents, photos, and code.
```bash
# Restore directly to the root (/) to put files back in /home/user/
sudo restic -r rclone:fortress-cloudbackup:linuxguardian/home restore latest \
    --target / \
    --password-file /path/to/your/saved/password.txt \
    --option rclone.program="rclone --config /home/$USER/.config/rclone/rclone.conf"
```

### Step 4: The "Environment" Restore (System Configs)
This brings back your Wi-Fi passwords, custom app settings, and system tweaks. **Do not restore this directly to `/`**; it may conflict with your new install.
1. Restore to a temporary folder:
   ```bash
   mkdir /tmp/old_system
   sudo restic -r rclone:fortress-cloudbackup:linuxguardian/system restore latest \
       --target /tmp/old_system \
       --password-file /path/to/your/saved/password.txt \
       --option rclone.program="rclone --config /home/$USER/.config/rclone/rclone.conf"
   ```
2. Manually copy critical configs (like `/etc/NetworkManager/system-connections/`) from `/tmp/old_system/etc/` to your live `/etc/`.

## 🏎️ Scenario 3: Migration to a New Machine
**Case:** You bought a new high-end PC and want it to look exactly like your old one.

1. Follow **Scenario 2** to restore your `/home` folder.
2. Clone your `Project LinuxGuardian` repository to the new machine.
3. Run the installer: `sudo ./install.sh`.
4. Run `sudo linuxguardian init` to re-enable the background timers.
5. **Result:** Your new machine will now continue the backup schedule exactly where the old one left off.
